# 用户视角功能验证

> **核心理念**：编译通过 ≠ 功能实现

## 前置条件
- 代码已完成（Implement 阶段通过）
- 单元测试和集成测试通过
- 文档已同步（Doc-Sync 阶段通过）

## 核心原则

### SMART 验收条件

验收条件必须遵循 SMART 原则：

| 原则 | 说明 | 示例 |
|------|------|------|
| **S**pecific | 具体的用户操作 | "输入正确用户名和密码，点击登录按钮" |
| **M**easurable | 可测量的结果 | "2秒内跳转到首页" |
| **A**chievable | 可实现的 | 不要写"永远不崩溃" |
| **R**elevant | 相关的用户价值 | "显示用户名，确认登录成功" |
| **T**ime-bound | 可测试的时间约束 | "登录态保持24小时" |

**反例**：
- ✗ "用户可以登录" — 太模糊
- ✗ "系统永远不崩溃" — 不可实现
- ✗ "页面好看" — 不可测量

**正例**：
- ✓ "用户输入正确用户名和密码，点击登录按钮，2秒内跳转到首页，显示用户名"
- ✓ "用户输入错误密码，显示'密码错误'提示，不跳转页面"
- ✓ "关闭浏览器后重新打开，24小时内无需重新登录"

## 流程

### 1. 列出用户验收条件

从功能规格文档中提取验收条件，转换为 SMART 检查项：

```markdown
## 用户验收条件

### 功能性验收
- [ ] AC-1: 用户输入正确用户名和密码，点击登录按钮，2秒内跳转到首页，显示用户名
- [ ] AC-2: 用户输入错误密码，显示"密码错误"提示，不跳转页面
- [ ] AC-3: 用户点击退出登录，跳转到登录页，清除登录态
- [ ] AC-4: 关闭浏览器后重新打开，24小时内无需重新登录

### 非功能性验收
- [ ] AC-5: 登录接口响应时间 < 500ms（P95）
- [ ] AC-6: 支持 Chrome/Firefox/Safari 最新版本
- [ ] AC-7: 支持移动端浏览器（iOS Safari, Android Chrome）
- [ ] AC-8: 错误信息清晰易懂，不暴露系统内部信息
```

### 2. 执行验证

#### 自动化验证（E2E 测试）

```bash
# 运行相关 E2E 测试
npx playwright test tests/e2e/login.spec.ts

# 查看测试结果
npx playwright show-report
```

#### 手动验证

对于无法自动化的场景，进行手动验证：

| AC | 验证项 | 操作步骤 | 预期结果 | 实际结果 |
|----|--------|---------|---------|---------|
| AC-1 | 登录成功 | 输入正确账号密码 | 2秒内跳转到首页 | — |
| AC-2 | 密码错误 | 输入错误密码 | 显示"密码错误"提示 | — |
| AC-7 | 移动端访问 | 手机浏览器访问 | 页面正常显示 | — |

### 3. 输出验证报告

```markdown
## 功能验证报告

### 验证概要
- 功能：用户认证模块
- 验证时间：2026-06-02
- 验证人：xxx

### 验证结果

| AC | 描述 | 自动化 | 手动 | 结果 |
|----|------|--------|------|------|
| AC-1 | 登录成功 | ✓ | — | PASS |
| AC-2 | 密码错误 | ✓ | — | PASS |
| AC-3 | 退出登录 | ✓ | — | PASS |
| AC-4 | 登录态保持 | ✗ | ✓ | PASS |
| AC-5 | 响应时间 | ✓ | — | PASS |
| AC-6 | 浏览器兼容 | — | ✓ | PASS |
| AC-7 | 移动端 | — | ✓ | FAIL |
| AC-8 | 错误信息 | — | ✓ | PASS |

### 问题记录

#### 问题 1：移动端页面显示异常
- **AC**: AC-7
- **现象**：iOS Safari 上按钮错位
- **预期**：按钮正常显示
- **影响**：移动端用户体验
- **建议**：检查 CSS 响应式布局

### 验证结论
- 通过：7/8
- 失败：1/8
- **建议**：修复移动端问题后重新验证
```

## Anti-Patterns

- ✗ "编译通过就算完成了" — 编译通过 ≠ 功能实现
- ✗ 只测 happy path，不测边界情况
- ✗ 跳过手动验证 — 有些场景无法自动化
- ✗ 验收条件模糊，无法验证 — "用户可以登录"太模糊
- ✗ 只测功能，不测用户体验 — 响应时间、错误提示
- ✗ 发现问题后不记录，直接跳过
