# 代码清理

## Overview

安全地清理死代码和未使用的文件。**每一步都必须验证，宁可漏删不可误删。**

## When to Use

- 清理死代码（未使用的函数、变量、import）
- 删除临时文件、调试代码
- 重构遗留代码

## Process

1. **分析阶段**
   - 搜索未使用的符号（函数、变量、类型、import）
   - 搜索临时文件（tmp、debug、test 输出）
   - 检查依赖关系图，确认删除影响范围

2. **确认阶段**
   - 列出待删除的项目清单
   - 向用户确认删除范围（特别是大范围清理）
   - 对不确定的项标注 "待确认"

3. **执行阶段**
   - 按依赖顺序删除（先删引用方，再删定义方）
   - 每批删除后验证编译/测试通过

4. **验证阶段**
   - 完整编译检查
   - 运行测试套件
   - 确认无残留引用

## Red Flags — STOP

- ❌ "这个肯定没用，直接删" → **必须确认无引用再删**
- ❌ "删完再验证" → **分批删除，每批验证**
- ❌ "测试太慢，跳过" → **不验证不提交**
- ❌ "一次全清理完" → **大范围清理需用户确认**

## Common Rationalizations

| 借口 | 现实 |
|------|------|
| "grep 没搜到就是没用" | grep 搜不到动态引用、反射、配置引用 |
| "旧代码肯定没人用" | 有些"旧代码"是被外部系统调用的 |
| "删错了再加回来" | 加回来需要理解当时的上下文，成本很高 |
| "测试太慢了" | 没测试验证的删除就是在赌 |

## Completion Checklist

- [ ] 待删除清单已列出
- [ ] 用户已确认删除范围（大范围清理时）
- [ ] 按依赖顺序删除
- [ ] 编译检查通过
- [ ] 测试套件通过
- [ ] 无残留引用
